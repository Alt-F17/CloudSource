# hackathon-starter

Next.js 15 + Auth.js v5 (Google OAuth) + Neon Postgres + Drizzle ORM + Tailwind + Shadcn/UI

Clone this, fill in 4 env vars, and you have a working authenticated app in under 10 minutes.

---

## stack

| layer | tool |
|---|---|
| framework | Next.js 15 (App Router) |
| auth | Auth.js v5 + Google OAuth |
| database | Neon (serverless Postgres) |
| orm | Drizzle ORM |
| ui | Tailwind CSS + Shadcn/UI |
| deploy | Vercel |

## setup (10 min)

### 1. clone and install

```bash
git clone https://github.com/YOUR_USERNAME/hackathon-starter.git
cd hackathon-starter
npm install
```

### 2. set up neon

1. Go to [neon.tech](https://neon.tech) and create a free account
2. Create a new project (takes 30 seconds)
3. Copy the connection string from the dashboard — looks like:
   `postgresql://user:password@ep-xxxx.us-east-2.aws.neon.tech/neondb?sslmode=require`

### 3. set up google oauth

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create a project > APIs & Services > Credentials > Create OAuth 2.0 Client ID
3. Application type: Web application
4. Authorized redirect URIs:
   - `http://localhost:3000/api/auth/callback/google` (dev)
   - `https://your-app.vercel.app/api/auth/callback/google` (prod, add after deploy)
5. Copy the Client ID and Client Secret

### 4. env vars

Create `.env.local` in the root:

```env
# neon
DATABASE_URL=postgresql://...your neon connection string...

# auth.js
AUTH_SECRET=your-random-secret-run-npx-auth-secret-to-generate

# google oauth
AUTH_GOOGLE_ID=your-google-client-id
AUTH_GOOGLE_SECRET=your-google-client-secret
```

Generate `AUTH_SECRET`:
```bash
npx auth secret
```

### 5. push schema and run

```bash
npm run db:push    # pushes schema to neon
npm run dev        # starts dev server
```

Open [http://localhost:3000](http://localhost:3000)

## key files explained

### `lib/auth.ts`
```ts
import NextAuth from "next-auth"
import Google from "next-auth/providers/google"
import { DrizzleAdapter } from "@auth/drizzle-adapter"
import { db } from "@/db"

export const { handlers, signIn, signOut, auth } = NextAuth({
  adapter: DrizzleAdapter(db),
  providers: [Google],
})
```

### `middleware.ts`
```ts
export { auth as middleware } from "@/lib/auth"

export const config = {
  matcher: ["/dashboard/:path*"],  // add any protected routes here
}
```

### `db/schema.ts`
```ts
import { pgTable, text, timestamp } from "drizzle-orm/pg-core"

// your app tables go here
export const trips = pgTable("trips", {
  id: text("id").primaryKey(),
  userId: text("user_id").notNull(),
  destination: text("destination").notNull(),
  createdAt: timestamp("created_at").defaultNow(),
})

// auth.js tables are auto-generated by DrizzleAdapter — don't define them manually
```

### `db/index.ts`
```ts
import { drizzle } from "drizzle-orm/neon-http"
import { neon } from "@neondatabase/serverless"
import * as schema from "./schema"

const sql = neon(process.env.DATABASE_URL!)
export const db = drizzle(sql, { schema })
```

## notes for PII / travel planner

- all user data lives in your own Neon DB — nothing stored by third parties
- Auth.js sessions are DB-backed (not JWT-only), so you can invalidate them
- use `auth()` in server components to get the current session:
  ```ts
  import { auth } from "@/lib/auth"
  const session = await auth()
  // session.user.id, session.user.email, session.user.name
  ```
- never log `session.user` to the console in prod — it contains PII
- add `?sslmode=require` to your Neon connection string (already in the example above)
